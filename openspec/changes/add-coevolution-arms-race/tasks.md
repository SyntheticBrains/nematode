## 5. Red Queen Metrics Module (PR 3)

- [x] 5.1 Create `quantumnematode/evolution/redqueen_metrics.py` with the five pure functions: `phenotypic_cycling`, `trait_escalation`, `fitness_lag`, `coupled_rate`, `generality`; all operate on numpy arrays / floats only (no Genome / FitnessFunction dependencies).
- [x] 5.2 Implement `phenotypic_cycling`: autocorrelation peak detection within bounded lag range, OLS-detrend before computing autocorr (rejects monotone ramps), permutation-null p-value (500 shuffles); return dict with `cycling_detected`, `dominant_period`, `p_value`. Tracks peak POSITIVE correlation (period repetition) rather than peak |corr| (which would peak at half-period for sine inputs). Note: chose autocorr-only over Lomb-Scargle for simplicity + zero scipy dep; sufficient for the M5 verdict-gate inference task.
- [x] 5.3 Implement `trait_escalation`: closed-form OLS over `gen_window=(5, 30)` with two-sided z-test on slope (asymptotic ŌĆö n in window up to 26, no scipy dep); return dict with `escalation_detected`, `slope`, `slope_sign`, `slope_se`, `p_value`. Zero-residual perfect-fit case returns p_value=0 if slope nonzero (slope is exactly known) rather than degenerate p=1.
- [x] 5.4 Implement `fitness_lag`: cross-correlation between two series with `max_lag` parameter; return best-lag scalar (peak POSITIVE correlation per phase-alignment convention; anti-phase belongs to `coupled_rate`) or NaN for zero-variance / max_lag-too-large inputs.
- [x] 5.5 Implement `coupled_rate`: per-generation delta computation followed by Pearson correlation of deltas via `np.corrcoef`; return scalar in [-1, 1] or NaN for zero-variance deltas.
- [x] 5.6 Implement `generality`: summarise (gens ├Ś opponents) held-out probe matrix via per-opponent OLS slope, normalise by max |slope| ŌåÆ scalar in [-1, 1]. +1 = uniform improvement; 0 = flat or anti-symmetric mix; negative = decline.
